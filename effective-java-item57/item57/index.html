Item57. 지역변수의 범위를 최소화하라

가장 처음 쓰일 때 선언하라

  • 미리 선언된 지역변수는 가독성을 떨어뜨린다.
  • 필요한 것보다 더 넓은 가용범위를 가질 경우 의도하지 않는 문제 발생 가능

선언과 동시에 초기화 하라

  • 단, try-cath문은 예외. 변수값을 try 바깥에서 사용한다면 try 블록 직전에 선언

try {

while문보다는 for문 사용

  • 변수 유효 범위를 for문 범위와 일치 시킬 수 있다.
// while문 사용할 경우
public static void main(String[] args) {
   ...
   Iterator<Element> i = c.iterator();
   while(i.hasNext()) {
      doSomething(i.next());
   }
   ...
   Iterator<Element> i2 = c2.iterator();
   while(i.hasNext()) { // 버그
      doSomething(i2.next());
   }
}

// for문 사용할 경우
public static void main(String[] args) {
     ...
   for(Iterator<Element> i = c.iterator(); i.hasNext();) {
      Element e = i.next();
   }

    // 컴파일 오류
   for(Iterator<Element> i2 = c2.iterator(); i.hasNext();) {
      Element e2 = i2.next();
   }
}

메서드를 작게 유지하고 한 가지 기능에 집중

  • 메서드가 한 가지 기능에 집중하면 지역변수의 범위도 줄어든다.
// 가격 제공 함수
double getPrice() {
   int basePrice = _quantity * _itemPrice;
   double discountFactor;
   if (basePrice > 1000) discountFactor = 0.95;
   else discountFactor = 0.98;
   return basePrice * discountFactor;
}
...
// 임시변수를 메서드 호출로 전환
int basePrice() {
   return _quantity * _itemPrice;
}

double discountFactor() {
   if (basePrice() > 1000) return 0.95;
   else return 0.98;
}

double getPrice() {
   return basePrice() * discountFactor();
}

*리팩토링 1판, 임시변수를 메서드 호출로 전환 발췌

  • 임시 변수 basePrice와 discountFactor를 메소드로 전환, 메소드를 기능별 분리
